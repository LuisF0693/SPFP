---
description: Padrão de Entrega e Explicação de Tarefas
---

Sempre que o usuário solicitar uma implementação ou correção:

1. **Planejamento**: Crie ou atualize o `implementation_plan.md`.
2. **Execução**: Realize as alterações no código seguindo o padrão visual "Premium".
3. **Documentação**: Atualize o `walkthrough.md` explicando o "O quê", "Por quê" e "Como".
// turbo-all
4. **Git**: Execute os comandos de commit e push para manter o servidor atualizado.
5. **Notificação**: Informe o usuário sobre a conclusão e peça para testar.
