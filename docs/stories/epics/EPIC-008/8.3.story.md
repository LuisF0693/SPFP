# Story 8.3 — Histórico de Conversas Navegável

## Status: Done

## Contexto
O usuário perde todo o histórico ao sair do Consultor IA. Apesar de os dados serem salvos no Supabase (Story 5.2), não há interface para navegar e retomar conversas anteriores.

## Acceptance Criteria

- [ ] AC1: Aba "Histórico" ao lado da aba "Chat" no Consultor IA
- [ ] AC2: Lista de sessões anteriores com: data, primeiro prompt enviado e número de mensagens
- [ ] AC3: Clicar em uma sessão carrega o histórico completo no chat
- [ ] AC4: Botão "Nova conversa" inicia sessão limpa sem perder o histórico
- [ ] AC5: Cada sessão tem botão de exclusão individual
- [ ] AC6: Sessões agrupadas por data (Hoje, Esta semana, Este mês, Anteriores)
- [ ] AC7: Máximo 50 sessões exibidas; sessões mais antigas podem ser carregadas

## Arquivos a Modificar

- `src/components/Insights.tsx` — abas e navegação de histórico
- `src/services/aiHistoryService.ts` — buscar sessões agrupadas
- Possível: `src/components/InsightHistory.tsx` — componente de histórico

## Tarefas

- [ ] Definir modelo de "sessão": `session_id` agrupando mensagens da mesma conversa
- [ ] Atualizar `aiHistoryService` para salvar `session_id` com cada mensagem
- [ ] Criar função `getSessionList(userId)` — retorna sessões agrupadas
- [ ] Implementar aba "Histórico" com lista de sessões
- [ ] Clicar na sessão carrega mensagens no chat
- [ ] Botão "Nova conversa" com geração de novo `session_id`
- [ ] Exclusão individual de sessão

- [ ] Commit: `feat: histórico de conversas navegável no Consultor IA (Story 8.3)`
