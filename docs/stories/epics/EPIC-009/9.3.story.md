# Story 9.3 — Boards dentro de cada Squad

## Status: Pending

## Contexto
Boards são os projetos/listas dentro de um Squad (equivalente a "Lists" no ClickUp). Ex: dentro do Squad Marketing existem os boards "Campanha Q1", "Conteúdo Orgânico", "Email Marketing". Esta story cria o CRUD de boards.

## Acceptance Criteria

- [ ] AC1: Dentro de cada Squad, área de boards com cards visuais
- [ ] AC2: Cada board tem: nome, descrição, cor, ícone e contagem de tasks
- [ ] AC3: Modal de criação/edição de board
- [ ] AC4: Clicar no board abre o Kanban de tasks (Story 9.4)
- [ ] AC5: Boards podem ser arquivados (não deletados)
- [ ] AC6: Boards ordenáveis por drag-and-drop na visão do squad
- [ ] AC7: Dados persistem no Supabase (tabela `company_boards`)

## Schema Supabase

```sql
CREATE TABLE company_boards (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  squad_id UUID REFERENCES company_squads(id) ON DELETE CASCADE,
  user_id UUID REFERENCES auth.users(id),
  name TEXT NOT NULL,
  description TEXT,
  icon TEXT DEFAULT '📋',
  color TEXT DEFAULT '#6366f1',
  is_archived BOOLEAN DEFAULT false,
  sort_order INT DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT now()
);
-- RLS: user_id = auth.uid()
```
