# Story 12.2 — Hotmart Webhook → Supabase (Cursos e Produtos Digitais)

## Status: Done

## Contexto
Integrar o Hotmart ao CRM Empresa via webhooks — cada compra de curso ou produto digital aparece no dashboard financeiro da empresa, com dados do aluno/comprador.

## Acceptance Criteria

- [ ] AC1: Supabase Edge Function receptora de webhooks do Hotmart
- [ ] AC2: Eventos processados: `PURCHASE_COMPLETE`, `PURCHASE_CANCELED`, `PURCHASE_REFUNDED`
- [ ] AC3: Cada evento salvo na tabela `company_revenue` (mesma tabela do Stripe — `source: 'hotmart'`)
- [ ] AC4: Dados do comprador: nome, email, produto, valor, data
- [ ] AC5: Validação de token do Hotmart no header
- [ ] AC6: Produtos do Hotmart mapeados para `company_products`

## Schema Adicional

```sql
CREATE TABLE company_products (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  source TEXT NOT NULL, -- 'stripe' | 'hotmart'
  external_product_id TEXT UNIQUE,
  name TEXT NOT NULL,
  price DECIMAL(10,2),
  type TEXT, -- 'subscription' | 'course' | 'one_time'
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMPTZ DEFAULT now()
);
```

## Dev Notes
Hotmart usa `hottok` como token de validação no header `X-Hotmart-Webhook-Token`. Documentação: [developers.hotmart.com/docs/en/webhooks]. Formato do payload diferente do Stripe — mapear campos `purchase.price.value`, `buyer.name`, `product.name`.
