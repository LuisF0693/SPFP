# Story 5.1 — Aposentadoria: Dados Salvos e Isolados por Cliente

## Status: Pending

## Contexto
Os dados inputados na aba de Aposentadoria (idade atual, renda, objetivo, etc.) precisam ser salvos por cliente no Supabase, não sendo compartilhados ou sobrescritos entre usuários.

## Acceptance Criteria

- [ ] AC1: Dados da aposentadoria (idade, renda mensal, objetivo, patrimônio atual, etc.) são salvos no Supabase vinculados ao `user_id`
- [ ] AC2: Ao fazer login, os dados da aposentadoria do usuário são carregados automaticamente
- [ ] AC3: Admin impersonando cliente X vê os dados de aposentadoria de X, não os do admin
- [ ] AC4: Ao encerrar impersonação, admin recupera seus próprios dados
- [ ] AC5: Dados não vazam entre sessões de usuários diferentes

## Arquivos a Modificar

- `src/components/Retirement.tsx` — estado local → Supabase
- `src/context/FinanceContext.tsx` — adicionar `retirementData` ao estado global
- Supabase: tabela `retirement_data` com `user_id`
