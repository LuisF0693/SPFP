# Story 5.1 — Aposentadoria: Dados Salvos e Isolados por Cliente

## Status: Done

## Contexto
Os dados inputados na aba de Aposentadoria (idade atual, renda, objetivo, etc.) precisam ser salvos por cliente no Supabase, não sendo compartilhados ou sobrescritos entre usuários.

## Acceptance Criteria

- [x] AC1: Dados da aposentadoria (idade, renda mensal, objetivo, patrimônio atual, etc.) são salvos no Supabase vinculados ao `user_id`
- [x] AC2: Ao fazer login, os dados da aposentadoria do usuário são carregados automaticamente
- [x] AC3: Admin impersonando cliente X vê os dados de aposentadoria de X, não os do admin
- [x] AC4: Ao encerrar impersonação, admin recupera seus próprios dados
- [x] AC5: Dados não vazam entre sessões de usuários diferentes
