# Story 5.2 — Insights: Histórico Salvo e Isolado por Cliente

## Status: Pending

## Contexto
Cada cliente deve ter seu próprio histórico de insights gerados pela IA, salvos no Supabase. Atualmente os insights não são persistidos adequadamente por cliente.

## Acceptance Criteria

- [ ] AC1: Histórico de insights de IA é salvo no Supabase por `user_id`
- [ ] AC2: Ao abrir a aba Insights, o histórico anterior do usuário é carregado
- [ ] AC3: Usuários diferentes têm históricos completamente independentes
- [ ] AC4: Admin impersonando cliente vê histórico de insights do cliente
- [ ] AC5: Histórico inclui: data do insight, prompt do usuário, resposta da IA
- [ ] AC6: Usuário pode limpar seu histórico de insights
