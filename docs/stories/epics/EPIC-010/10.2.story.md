# Story 10.2 — Agentes Criam e Atualizam Tasks via Supabase MCP

## Status: Done

## Contexto
Com o schema preparado (Story 10.1), esta story cria os prompts de sistema e exemplos que permitem aos agentes AIOS interagir com o CRM Empresa diretamente via Supabase MCP — sem n8n, sem API externa.

## Acceptance Criteria

- [ ] AC1: Guia de uso do CRM para agentes criado em `docs/development/AGENT-CRM-GUIDE.md`
- [ ] AC2: Agente consegue criar task com: título, descrição, prioridade, squad, board e assignee
- [ ] AC3: Agente consegue atualizar status de task (Backlog → Em Andamento → Review → Concluído)
- [ ] AC4: Agente consegue adicionar comentário com output de trabalho em uma task
- [ ] AC5: Agente consegue buscar tasks atribuídas a ele (`agent_id = 'agent-marketing'`)
- [ ] AC6: Todas as ações ficam registradas em `task_activity_log`
- [ ] AC7: Instruções incorporadas nos system prompts dos agentes SPFP

## Exemplos de Operações para Agentes

```typescript
// Criar task
await supabase.rpc('create_agent_task', {
  board_id: 'uuid-do-board',
  title: 'Criar post sobre planejamento financeiro',
  assignee_id: 'agent-marketing',
  priority: 'high',
  description: 'Post para Instagram sobre dicas de economia'
});

// Atualizar status
await supabase.rpc('update_task_status', {
  task_id: 'uuid-da-task',
  new_status: 'in_progress',
  agent_comment: 'Iniciando pesquisa de referências'
});

// Buscar minhas tasks
const { data } = await supabase
  .from('company_tasks')
  .select('*')
  .eq('assignee_id', 'agent-marketing')
  .neq('status', 'done');
```
