# Story 4.2 — Edição em Lote de Múltiplos Lançamentos

## Status: Done

## Contexto
Atualmente o usuário só pode editar um lançamento por vez. Quando precisa reclassificar várias transações (ex: mudar categoria, mover para outra conta ou trocar cartão), precisa fazer uma a uma. A feature de edição em lote resolve isso.

## Acceptance Criteria

- [x] AC1: Na lista de lançamentos, checkboxes aparecem ao lado de cada transação ao ativar modo de seleção
- [x] AC2: Usuário pode selecionar múltiplos lançamentos com checkbox
- [x] AC3: Ao ter 2+ selecionados, aparece barra de ação em lote com opções: Categoria, Conta, Cartão
- [x] AC4: Ao clicar em "Categoria", abre modal com lista visual de categorias — aplica em todos selecionados
- [x] AC5: Ao clicar em "Conta", abre modal com lista de contas — aplica em todos selecionados
- [x] AC6: Ao clicar em "Cartão", abre modal com lista de cartões de crédito — aplica em todos selecionados
- [x] AC7: Toast verde de confirmação com quantidade e destino após aplicar
- [x] AC8: Operação em lote usa updateTransactions (salva via Supabase + localStorage)
- [x] AC9: Botão "Todos do mês" na barra para selecionar/desmarcar todos do filtro atual
- [x] AC10: Botão X para cancelar seleção e sair do modo lote
