# Story 4.1 — BUG FIX: Edição de Parcelado/Fixo Mensal Não Exclui Original

## Status: Done

## Contexto
Quando o usuário edita uma transação parcelada ou fixa mensal e altera a data para o mês seguinte, o sistema **cria a nova transação no mês correto** mas **não exclui a transação original**. Resultado: duplicação indesejada de lançamentos.

## Problema Reportado
- Usuário edita compra parcelada/fixa mensal
- Altera data para mês seguinte
- Sistema cria nova no mês seguinte ✅
- Sistema **não** exclui a do mês original ❌
- Resultado: duplicação do lançamento

## Acceptance Criteria

- [x] AC1: Ao editar transação parcelada e alterar a data para mês diferente, a transação original é atualizada (não duplicada)
- [x] AC2: Ao editar transação fixa mensal e alterar a data para mês diferente, a transação original é atualizada (não duplicada)
- [x] AC3: A nova transação é criada com todos os dados editados (valor, categoria, conta, data)
- [x] AC4: Apenas a transação editada é movida (groupId preservado, outras parcelas não afetadas)
- [x] AC5: O comportamento deve ser consistente para transações no Supabase e no localStorage
- [x] AC6: Nenhuma regressão em outras funcionalidades de edição de transações
