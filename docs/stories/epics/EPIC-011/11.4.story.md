# Story 11.4 — YouTube Data API (Upload de Vídeos)

## Status: Pending

## Contexto
Integrar o Marketing Hub com YouTube Data API v3 para upload de vídeos diretamente do SPFP. Vídeos são salvos no Supabase Storage e uploadados para o YouTube pelo próprio sistema.

## Acceptance Criteria

- [ ] AC1: Configuração de credenciais YouTube na tela de Settings (OAuth2 Client ID + Secret)
- [ ] AC2: Upload de vídeo para YouTube com: título, descrição, tags, thumbnail, categoria, privacidade
- [ ] AC3: Barra de progresso durante upload (YouTube API suporta resumable uploads)
- [ ] AC4: Status pós-upload: "Processando" → "Publicado" com link do vídeo
- [ ] AC5: Thumbnail personalizada (imagem) ou gerada automaticamente pelo YouTube
- [ ] AC6: Opção de privacidade: Público, Não-listado, Privado
- [ ] AC7: Log de uploads com status, ID do vídeo e link

## Fluxo Técnico

```
Frontend SPFP
    ↓ (envia content_id)
Supabase Edge Function: publish-youtube
    ↓ (busca vídeo do Storage + credenciais)
Google OAuth2 Token Refresh
    ↓ access_token válido
YouTube Data API v3:
  POST /upload/youtube/v3/videos (resumable upload)
    ↓ video_id
  Status polling até "processed"
    ↓
Supabase: atualiza published_urls com YouTube URL
```

## Dev Notes
YouTube requer OAuth2 (não API Key simples) — fluxo de autorização necessário. Refresh token tem validade longa se escopo `youtube.upload` for solicitado. Vídeos grandes: usar resumable upload da YouTube API. Polling de status: `GET /videos?id={videoId}&part=status` a cada 30s.
