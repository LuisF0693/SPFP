# 2.1: Ver Escritório Virtual com 4 Departamentos

**Status:** Draft
**Sprint:** 5
**Points:** 13
**Epic:** EPIC-002

---

## User Story

**Como** empresário
**Quero** ver minha empresa como um escritório virtual com 4 departamentos distintos
**Para que** eu tenha uma visão clara e engajadora da estrutura organizacional

---

## Description

Implementar a interface visual 2D top-down do escritório virtual mostrando 4 departamentos (Financeiro, Marketing, Operacional, Comercial) com áreas visualmente distintas, personagens NPC estáticos representando os agentes, labels descritivos e interatividade básica (hover e clique). O mapa deve ser responsivo e acessível, renderizado em tela cheia ou área principal.

---

## Acceptance Criteria

### AC-001.1: Mapa Renderizado em Tela Cheia
- [ ] Mapa ocupa 100% da altura e largura disponível
- [ ] Sem overflow ou scrollbars desnecessários
- [ ] Layout mantém proporções em diferentes resoluções

### AC-001.2: 4 Áreas Visíveis
- [ ] Área Financeiro (canto superior esquerdo)
- [ ] Área Marketing (canto superior direito)
- [ ] Área Operacional (canto inferior esquerdo)
- [ ] Área Comercial (canto inferior direito)

### AC-001.3: Visual Distinto por Departamento
- [ ] Financeiro: cor emerald (#10B981), elementos de gráficos
- [ ] Marketing: cor violet (#8B5CF6), elementos criativos
- [ ] Operacional: cor amber (#F59E0B), elementos de organização
- [ ] Comercial: cor blue (#3B82F6), elementos de negociação

### AC-001.4: Personagens NPC Estáticos em Cada Departamento
- [ ] CFO no Financeiro (terno, com gráficos)
- [ ] CMO no Marketing (casual criativo, com post-its)
- [ ] COO no Operacional (organizado, com checklist)
- [ ] CSO no Comercial (com headset, telefone)

### AC-001.5: Labels Identificando Cada Departamento
- [ ] Label "FINANCEIRO" visível na área Financeiro
- [ ] Label "MARKETING" visível na área Marketing
- [ ] Label "OPERACIONAL" visível na área Operacional
- [ ] Label "COMERCIAL" visível na área Comercial

### AC-001.6: Hover em Departamento Mostra Highlight
- [ ] Cursor muda para pointer ao passar sobre departamento
- [ ] Área fica com destaque visual (sombra, borda, opacidade)
- [ ] Transição suave sem lag

### AC-001.7: Clique em Departamento Abre Modal/Drawer
- [ ] Clique abre modal ou drawer com dashboard respectivo
- [ ] Modal possui botão de fechar (X)
- [ ] Clique fora do modal fecha (se drawer lateral)

### AC-001.8: Responsivo em Diferentes Tamanhos
- [ ] Desktop: Layout completo 2x2
- [ ] Tablet: Layout ajustado com espaçamento reduzido
- [ ] Mobile: Pode usar layout alternativo (cards stacked ou tabs)

## Scope

### IN (Incluído)
- Interface visual 2D top-down do escritório
- 4 departamentos com cores e estilos distintos
- Personagens NPC estáticos (emojis ou ícones)
- Labels descritivos
- Interatividade hover e clique
- Design system com paleta de cores
- Responsividade (desktop/tablet/mobile)
- Acessibilidade (navegação por teclado, labels aria)

### OUT (Não Incluído)
- Avatar do usuário que se move
- Animações de personagens caminhando
- Visualização real-time de agentes Claude Code
- Customização do mapa pelo usuário
- Multiplayer (ver outros usuários)
- Integração com dados reais (vêm nos dashboards)

## Technical Notes

### Implementation Approach

1. **Layout Base** (`src/components/CorporateHQ.tsx`)
   - Grid 2x2 usando CSS Grid ou Flex
   - Cada área é uma sub-seção clicável

2. **Design** (`src/styles/corporateDesignSystem.ts`)
   - Paleta: Financeiro #10B981, Marketing #8B5CF6, Operacional #F59E0B, Comercial #3B82F6
   - Background geral: #0F172A (dark blue)
   - Spacing, sombras, transições

3. **Componentes**
   - `CorporateHQ.tsx` (NEW) - container principal
   - `DepartmentArea.tsx` (NEW) - área individual com personagem e label
   - `NPCCharacter.tsx` (NEW) - renderização do personagem

4. **Libraries**
   - TailwindCSS (layout, responsividade)
   - React Icons (Lucide) para elementos
   - framer-motion (transições hover suave)

5. **File Structure**
```
src/components/
├── corporate/
│   ├── CorporateHQ.tsx (NEW)
│   ├── DepartmentArea.tsx (NEW)
│   └── NPCCharacter.tsx (NEW)
└── styles/
    └── corporateDesignSystem.ts (NEW)
```
