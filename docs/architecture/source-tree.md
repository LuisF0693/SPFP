## File Placement Rules

### Documentation Files

| File Pattern | Location | Purpose |
|--------------|----------|---------|
| `*.md` (project docs) | `docs/` | Project documentation |
| Architecture docs | `docs/architecture/` | Technical standards |

### Source Code

| File Pattern | Location | Purpose |
|--------------|----------|---------|
| `*.tsx`, `*.ts` | `src/` | Application source code |
| `*.tsx` (components) | `src/components/` | UI components |
| `*.ts` (services) | `src/services/` | Logic and services |

### Configuration Files

| File | Location | Purpose |
|------|----------|---------|
| `.aios-core/core-config.yaml` | Root | AIOS configuration |
| `.env` | Root | Environment variables |
| `package.json` | Root | Dependencies |
| `tsconfig.json` | Root | TypeScript configuration |

## Anti-Patterns (Files NOT Allowed at Root)

- Do not place `.md` files at the root (except README.md).
- Avoid putting large amounts of logic in `App.tsx`; use services.

## Naming Conventions

### Directories
- Use **kebab-case** for directory names: `my-feature/`

### Files
- Source files: **kebab-case** or **PascalCase** for components.
- React components: **PascalCase** - `UserProfile.tsx`
- Services/Utils: **camelCase** - `aiService.ts`
