# Template: Landing Page Blueprint

**Template ID:** landing-page-blueprint
**Version:** 1.0
**Purpose:** Standard structure and sections for creating high-conversion landing pages

## Overview

This blueprint provides the foundational structure for any landing page in the squad. Use this as a starting point for architecture and design phases.

## Standard Section Order

### 1. Navigation / Header
- **Purpose:** Site navigation and branding
- **Elements:**
  - Logo (clickable, returns to home if multi-page)
  - Main navigation menu (3-5 items)
  - Secondary CTA button (e.g., "Sign Up" or "Demo")
  - Mobile menu toggle (hamburger)
- **Best Practices:**
  - Sticky navigation on scroll (optional)
  - Mobile optimization (vertical menu)
  - Avoid cluttering with too many options

---

### 2. Hero Section
- **Purpose:** Immediate value proposition and attention capture
- **Elements:**
  - Attention-grabbing headline (benefit-focused)
  - Subheadline (clarifies/expands on main headline)
  - Hero image/video (high quality, relevant)
  - Primary CTA button (prominent, action-oriented)
  - Optional: Social proof (e.g., "Trusted by 500+ companies")
- **Best Practices:**
  - Headline must communicate value in <10 seconds
  - Subheadline speaks to pain point or benefit
  - Image/video shows product in action
  - CTA button uses action verb (Download, Get Started, etc.)

---

### 3. Problem / Pain Point Section
- **Purpose:** Establish relevance and empathy
- **Elements:**
  - Section headline (acknowledge problem)
  - 2-4 bullet points describing pain points
  - Illustrative image or icon
  - Optional: Statistics showing problem prevalence
- **Best Practices:**
  - Use user language, not marketing jargon
  - Show you understand their struggle
  - Build case for why this matters

---

### 4. Solution / Features Section
- **Purpose:** Explain how product solves the problem
- **Elements:**
  - Section headline
  - 3-6 feature cards (each with: icon, title, description)
  - Accompanying images or demo video
  - Optional: Brief explanation of how each feature solves a pain point
- **Best Practices:**
  - Features = benefits, not specs
  - Use icons to aid scanning
  - One feature per card for clarity
  - Visuals on every card (screenshot, illustration, video)

---

### 5. How It Works / Use Case Section
- **Purpose:** Show practical application and ease of use
- **Elements:**
  - Section headline
  - 3-5 step process (numbered)
  - Visual for each step (screenshot, illustration, video)
  - Brief description of each step
  - Optional: Real user example or workflow diagram
- **Best Practices:**
  - Keep steps simple and understandable
  - Visuals should clearly show the step
  - Use timeline or numbered format
  - Time estimate if relevant (e.g., "Takes 5 minutes to setup")

---

### 6. Social Proof / Testimonials Section
- **Purpose:** Build trust through external validation
- **Elements:**
  - Section headline
  - 3-5 testimonial cards (each with: quote, author photo, name, title, company)
  - Optional: Star rating or rating scale
  - Optional: Customer logos/company names
  - Optional: Key metric (e.g., "Rated 4.9/5 by 500 users")
- **Best Practices:**
  - Use real quotes and photos (not stock images)
  - Include specific results when possible
  - Show variety of company sizes/types
  - Customer logos add credibility
  - Rotate testimonials if limited space

---

### 7. Results / Impact Section
- **Purpose:** Quantify the value delivered
- **Elements:**
  - Section headline
  - 3-4 key metrics (large numbers with context)
  - Metric format: [Number] [Unit] [Benefit Description]
  - Optional: Before/after comparison
  - Optional: Customer success stories
- **Best Practices:**
  - Use concrete numbers, avoid vague claims
  - Include time frame if relevant
  - Show variety of impact types (efficiency, revenue, time saved)
  - Make numbers visually prominent

---

### 8. Pricing Section (If Applicable)
- **Purpose:** Explain pricing structure and options
- **Elements:**
  - Section headline
  - 2-3 pricing tier cards (Starter, Pro, Enterprise)
  - For each tier: price, billing period, included features, CTA button
  - Optional: Pricing comparison table
  - Optional: FAQ callout
- **Best Practices:**
  - Highlight recommended tier (using badge or highlight)
  - Make most popular tier visually prominent
  - Include CTA button on each tier
  - Show clear feature differences
  - Offer annual discount if available

---

### 9. FAQ Section
- **Purpose:** Address common objections and questions
- **Elements:**
  - Section headline
  - 6-10 FAQ items in accordion format
  - Question (clickable header)
  - Answer (expanded on click)
  - Optional: Categories for grouping (Setup, Billing, Support, etc.)
- **Best Practices:**
  - Answer the questions users actually ask
  - Use user language in questions
  - Keep answers concise but complete
  - Address pricing, support, and integration FAQs
  - Expandable accordion saves space

---

### 10. Final CTA Section
- **Purpose:** Final conversion opportunity before footer
- **Elements:**
  - Compelling headline (urgency or benefit-focused)
  - Brief supporting copy (1-2 sentences)
  - Primary CTA button (same or similar to hero)
  - Optional: Secondary CTA (e.g., "Schedule Demo")
  - Optional: Limited-time offer or incentive
  - Optional: Badge showing value (e.g., "Free for 14 days")
- **Best Practices:**
  - Similar or same copy as hero CTA
  - Reinforce main benefit
  - Create sense of urgency if appropriate
  - Make button prominent and clickable

---

### 11. Trust Elements (Distributed)
- **Purpose:** Build confidence throughout the page
- **Elements (pick 3-5):**
  - Security badges (SSL, GDPR, etc.)
  - Partnership logos
  - Certification badges
  - Money-back guarantee badge
  - Free trial offer
  - "No credit card required" messaging
- **Best Practices:**
  - Place in logical sections (e.g., guarantee near CTA)
  - Use only authentic badges/certifications
  - Keep number limited (don't overwhelm)
  - Position near trust-critical sections (signup forms, pricing)

---

### 12. Footer
- **Purpose:** Navigation, legal, and additional resources
- **Elements:**
  - Company logo
  - Footer navigation (4-5 sections)
  - Contact information (email, phone)
  - Social media links
  - Legal links (Privacy, Terms, etc.)
  - Copyright notice
  - Optional: Newsletter signup
  - Optional: Address/office info
- **Best Practices:**
  - Organize links into categories
  - Include main navigation for non-foot visitors
  - Easy-to-find contact information
  - Mobile-optimized multi-column layout

---

## Mobile Optimization Notes

**Key Considerations:**
- Stack all sections vertically
- Increase button sizes (44x44px minimum)
- Reduce features to 2-3 per row
- Simplify tables to card format
- Ensure text is readable (font size >= 14px)
- Hide non-essential elements or move below fold
- Single-column testimonials
- Full-width CTAs

---

## Responsive Breakpoints

- **Mobile:** 320px - 767px
- **Tablet:** 768px - 1023px
- **Desktop:** 1024px+

---

## Color Strategy

**Recommended Colors:**
- **Primary:** Brand color (use for CTAs, headlines)
- **Secondary:** Accent color (supporting elements)
- **Neutral:** Gray scale (backgrounds, text)
- **Success:** Green (positive, confirmation)
- **Warning:** Orange/Yellow (caution, attention)
- **Error:** Red (errors, warnings)

**Contrast Requirements:**
- Text on background: 4.5:1 contrast ratio
- Large text (18pt+): 3:1 contrast ratio
- WCAG AA compliance minimum

---

## Typography Hierarchy

| Element | Font Size | Weight | Usage |
|---------|-----------|--------|-------|
| H1 (Main Headline) | 48px+ | Bold/700 | Hero section |
| H2 (Section Headline) | 32-40px | Bold/700 | Section headers |
| H3 (Card Headline) | 24px | Semi-bold/600 | Cards, features |
| Body Text | 16px | Regular/400 | Paragraphs, copy |
| Small Text | 12-14px | Regular/400 | Captions, footer |
| CTA Button | 16px | Semi-bold/600 | Interactive |

---

## Spacing Guidelines

- **Container margin:** 40px (desktop), 20px (mobile)
- **Section padding:** 60px top/bottom (desktop), 40px (mobile)
- **Card spacing:** 20px gap between cards (desktop), 16px (mobile)
- **Line height:** 1.6 for body text, 1.3 for headlines

---

## Form Best Practices

**Form Fields:**
- One field per line on mobile
- 2 columns on tablet/desktop
- Clear labels above fields
- Placeholder text for guidance
- Input validation with friendly error messages
- Minimum field size: 44px height

**CTA Button (Form Submit):**
- High contrast color (primary color)
- Clear action text (e.g., "Get Started", "Submit")
- Hover/active states clearly defined
- Loading state (spinner or text)
- Success confirmation after submit

**Privacy/Trust:**
- Mention data protection (e.g., "We respect your privacy")
- Link to Privacy Policy near form
- Optional: GDPR/CCPA compliance mention
- Trust badges near form

---

## Content Length Guidelines

| Section | Recommended Length |
|---------|-------------------|
| Headline | 6-12 words |
| Subheadline | 12-20 words |
| Body paragraph | 50-100 words |
| Feature description | 20-40 words |
| Testimonial | 100-150 words |
| FAQ answer | 50-150 words |

---

## Animation & Interaction Guidelines

**Recommended:**
- Subtle scroll-triggered animations (fade, slide)
- Button hover states
- Form field focus states
- Smooth transitions between sections

**Avoid:**
- Auto-playing videos (audio off, or user-triggered)
- Excessive animations (distracting)
- Flashing or strobing effects
- Parallax on scroll (performance/accessibility issues)

---

## SEO Optimization

- **Title Tag:** Include primary keyword, 50-60 characters
- **Meta Description:** Include call to action, 150-160 characters
- **H1 Tag:** One per page, includes primary keyword
- **H2 Tags:** Natural keyword usage, multiple allowed
- **Image Alt Text:** Descriptive, includes keyword when natural
- **Internal Links:** Link to related resources, use descriptive anchor text
- **Schema Markup:** Product, LocalBusiness, or Organization schema

---

## Accessibility Checklist

- [ ] Color contrast 4.5:1 (normal text), 3:1 (large text)
- [ ] All images have descriptive alt text
- [ ] Form labels properly associated with fields
- [ ] Focus indicator visible on all interactive elements
- [ ] Keyboard navigation functional (no mouse required)
- [ ] Video has captions/transcript
- [ ] No auto-playing sound
- [ ] Text readable without color alone
- [ ] Sufficient spacing between interactive elements

---

## Performance Targets

- **First Contentful Paint (FCP):** < 1.8s
- **Largest Contentful Paint (LCP):** < 2.5s
- **Cumulative Layout Shift (CLS):** < 0.1
- **Bundle Size:** < 200KB (gzipped)
- **Lighthouse Score:** >= 90

## Usage Instructions

1. Use this blueprint as a starting point for architecture
2. Adapt sections based on your specific product/service
3. Not all sections are required - use what's appropriate
4. Adjust order based on user journey and value proposition
5. Test section order with A/B testing to optimize
6. Maintain spacing, typography, and color guidelines
7. Ensure mobile responsiveness throughout
8. Validate accessibility before launch
