agent: research-analyst
inputs:
  - name: criativos-winners
    description: Criativos e conteúdos vencedores encontrados no monitoramento de concorrentes e tendências
outputs:
  - description: Swipe file organizado no Notion alimentando COPY e Media Buyer com referências
ferramentas: [Notion, Google Drive]
---

## O que faz
- Salva criativos vencedores de concorrentes e referências do nicho
- Organiza por categoria (hook, formato, oferta, CTA, tipo de conteúdo)
- Anota por que cada criativo funciona (análise breve do Research Analyst)
- Disponibiliza para COPY (escrever) e Media Buyer (replicar estrutura)
- Atualiza semanalmente com novos winners encontrados
- Remove criativos desatualizados (> 6 meses sem relevância)

## Não faz
- Copiar criativos literalmente para usar no SPFP
- Criar conteúdo diretamente a partir do swipe file
- Usar referências sem adaptar ao contexto SPFP

## Ferramentas
- Notion (swipe file principal)
- Google Drive (vídeos que precisam ser salvos)

## Estrutura do Swipe File no Notion

```
📁 Swipe File SPFP
├── 🎣 Hooks que funcionam
│   ├── Hooks de problema ("Você ainda...")
│   ├── Hooks de resultado ("Em 30 dias eu...")
│   └── Hooks de curiosidade ("O segredo que...")
├── 📐 Formatos vencedores
│   ├── Reels/TikTok < 15s
│   ├── Carrossel Instagram
│   └── Vídeo longo YouTube
├── 💰 Ofertas e CTAs
│   ├── Free trial
│   ├── Garantia
│   └── Urgência
├── 🏆 Anúncios pagos
│   ├── Meta Ads winners
│   └── Google Ads winners
└── 🔍 Concorrentes
    ├── Mobills
    └── GuiaBolso
```

## Como documentar cada item

```markdown
### [Nome/Descrição do criativo]
- **Fonte**: [concorrente / influenciador / nicho]
- **Plataforma**: [TikTok / Instagram / Facebook]
- **Por que funciona**: [análise de 1-2 linhas]
- **Aplicação no SPFP**: [como adaptar]
- **Salvo em**: [link do arquivo]
```
